# let, const 키워드와 블록 레벨 스코프

let 키워드

1. 변수 중복 선언 금지 <br>
   -> let 키워드로 이름이 같은 변수르 중복 선언하면 문법에러가 발생한다.

2. 블록 레벨 스코프<br>
   -> var 키워드로 선언한 변수는 오로지 함수의 코드 블록만을 지역 스콮프로 인정하는 함수 레벨 스코프를 따른다. 하지만 let 키워드로 선언한 변수는 모든 코드 블록(함수, if문, for문, while문 , try/catch문) 을 지역 스코프로 인정하는 블록 레벨 스코프를 ㄸ따른다.

- let 키워드로 선언한 변수는 "선언 단계" 와 "초기화 단계"가 분리되어 진행된다. 런타임 이전 자바스크립트 엔진에 의해 암묵적으로 선언 단계가 먼저 실행되지만 초기화 단계는가 먼저 실행되지만 초기화 단계는 변수 선언문에 도달 했을떄 실행 된다!

const 키워드<br>

- const 키워드는 상수를 선언하기 위해 사용한다. 하지만 반드시 상수만을 위해 사용하지는 않는다.

1. 선언과 초기화<br>
   -> const 키워드로 선언한 변수는 반드시 선언과 동시에 초기화해야한다.
2. 재할당 금지 <br>
   -> var 또는 let 키워드로 선언한 변수는 재할당이 자ㅏ유로우나 const 키워드로 선언한 변수는 재할당이 금지된다.
3. 상수 <br>
   -> 변수의 상대 개념인 상수는 재할당이 금지된 변수를 말한다.
   -> const 키워드로 선언된 변수는 재할당이 금지된다. CONST 키워드로 선언된 변수에 원시 값을 할당한 경우 원시 값은 변경할 수 업ㅂㅅ는 값이고 const 키워드에 의해 재할당이 금지되므로 할당된 값을 변경ㅎ할수 있는 방법은 없다.
