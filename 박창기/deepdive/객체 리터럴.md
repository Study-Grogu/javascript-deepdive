# 객체 리터럴

# 10.1 \_ 객체?

- 자바스크립트는 객체 기반의 프로그래밍 언어!
- 자바스크립트를 구성하는 거의 "모든것"이 객체다 .

-> 원시 타입은 단 하나의 값만 나타내지만 객체 타입은 다양한 타입의 값(원시 값 또는 다른 객체)을 하나의 단위로 구성한 복합적인 자료구조다.
<br/> or 원시타입의 값, 즉 원시 값은 변경 불가능 한 값이지만 객체타입의 값, 즉 객체는 변경 가능한값이다.

---

<br/>
-> 객체는 프로퍼티와 메서드로 구성된 집합체!

- 프로퍼티 : 객체의 상태를 나타내는 값(data)
- 메서드 : 프로퍼티(상태데이터)를 참조하고 조작할 수 있는 동작(behavior)

---

<br>

리터럴 (literal) : 사람이 이해할 수 있는 무자 또는 약속된 기호를 사용하여 값을 생성 하는 표기법

객체리터럴은 중괄호 ({...}) 내에 0개 이상의 프로퍼티를 정의한다.

---

<br>
자바스크립의 함수는 객체(일급 객체) , 함수는 값으로 취급할 수 있기 때문에 프로퍼티 값으로 사용할수 있다.

프로퍼티 값이 함수일 경우 일반 함수와 구분하기 위해 메서드(method)라고 부른다.
<br>
-> 메스드는 객체에 묶여 있는 함수를 의미한다.

---

<br>
프로퍼티 동적 생성

```
var = person = {
    name : 'lee'
}

person.age = 20;
console.log(person);  //{name : "lee" , age : 20}

```

프로퍼티 삭제

```
var person = {
    name : "lee"
}

//프로퍼티 생성
person.age = 20
// delete 연산자로 age 프로퍼티를 삭제할 수 있다.
delete person .age

console.log(person) //{name : "Lee"}

```

# 원시 값과 객체의 비교

<br>

- 원시타입의 값, 즉 원시 값은 변경 불가능한 값이다.
- 객체(참조) 타입의 값 객체는 변경 가능한 값

- 원시 값을 변수에 할당 하면 변수(확보된 메모리 공간) 에는 실제 값이 저장된다.
- 객체를 변수에 할당하면 변수(확보된 메모리 공간) 에는 참조 값이 저장된다.
  <br>

원시 값을 갖는 변수를 다른 변수에 할당하면 원본의 원시 값이 복사되어 전달 -> 값에 의한 전달

## 원시의 값

- 원시 타입의 값 즉 원시 값은 변경 불가능한 값! -> 읽기 전용값으로서 변경 x

- 결국은 두변수의 원시 값은 서로 다른 메모리 공간에 저장된 별개의 값이 되어 있는 한쪽에서 재할당을 통해 값을 변경하더라도 서로 간섭할수 없다.!

## 유사 배열 객체

- 유사 배열 객체란 마치 배열처럼 인덱스로 프로퍼티 값에 접근할 수 있고 length 프로퍼티를 갖는 객체를 말한다. 문자열은 마치 배열처럼 인덱스를 통해 각 문자에 접근할 수 있으며, length 프로퍼티를 갖기 때문에 유사배열 객체이고 for 문으로 순회 할수도있다.

```
var str = "string"

//문자열은 유사 배열이므로 배열과 유사하게 인덱스를 사용해 각 문자에 접근
console.log(str[0]); //s

console.log(str.length) //6
console.log(str.toUpperCase()) //string
```

## 객체

객체는 프로퍼티의 개수가 정해져 있지 않으며, 동적으로 추가괴고 삭제할 수 있다. 또한 프로퍼티의 값에도 제약이 없다. 따라서 객체는 원시 값과 같이 확보해야 할 메모리 공간의 크기를 사전에 정해 둘수 없다.

-> 객체는 복합적인 자료구조이다.

-> 객체(참조) 타입의 값, 즉 객체는 변경 가능한 값이다!

-> 원시 값은 변경 불가능한 값이므로 원시 값을 갖는 변수의 값을 변경하려면 재할당 외에는 방법이 없다. 하지만 객체는 변경 가능한 값이다. 따라서 객체를 할당한 변수는 재할당 없이 객체를 직접 변경할 수 있다. 즉 재할당 없이 프로퍼티를 동적으로 추가할 수도 있고 프로퍼티 값을 개신할 수도 있으며 프로퍼티 자체를 삭제할수도 있다!

메모리를 효율적으로 사용 or 비용절약 선능을 향상
-> 객체는 변경 가능한 값으로 설계
