# 10~11. 객체와 객체 값 비교

## 1. 객체란 무엇인가?

자바스크립트에서 객체는 여러 속성(property)과 메서드(method)를 하나의 단위로 묶어 관리할 수 있는 데이터 구조이다. 객체는 `key: value` 쌍으로 이루어져 있으며, `value`는 원시 타입이 될 수도 있고, 다른 객체가 될 수도 있다.

```javascript
// 객체 리터럴을 이용한 객체 생성
const person = {
  name: "김철수",
  age: 30,
  isStudent: false,
  greet: function () {
    console.log(`안녕하세요, 제 이름은 ${this.name}입니다.`);
  },
};

console.log(person.name); // 김철수
person.greet(); // 안녕하세요, 제 이름은 김철수입니다.
```

자바스크립트는 프로토타입 기반 객체지향 언어로 다양한 객체 생성 방법을 제공한다. 이 중 위와 같은 객체 리터럴 방식이 가장 보편적으로 사용되는 방식이다.

- 객체는 프로퍼티의 집합이며 프로퍼티는 키와 값으로 구성된다. 위 `person` 객체의 경우 `name`가 키, `"김철수"`가 값이 된다. 각 프로퍼티는 쉼표로 구분하며, 키와 값은 콜론으로 구분한다. 키는 보편적으로 문자열이며 식별자 규칙을 따르지 않을 경우 따옴표로 묶어야한다.
- 자바스크립트에서 함수는 일급 객체다. 객체에서 함수를 값으로 가지는 프로퍼티를 메서드라고 한다.
- 객체의 프로퍼티에는 `객체이름.프로퍼티이름` 또는 `객체이름["프로퍼티이름"]`의 형태로 접근한다.
- 프로퍼티는 값 갱신, 동적 생성, 삭제를 할 수 있다.

## 2. 객체의 참조(Reference) 특성

자바스크립트에서 객체는 메모리에 저장될 때 **값 자체가 아닌 메모리 주소(참조)를 저장**한다. 변수에 객체를 할당할 때, 실제 객체 데이터가 복사되는 것이 아니라 객체가 저장된 메모리 주소가 복사된다.

```javascript
const obj1 = { value: 10 };
const obj2 = obj1; // obj2는 obj1과 동일한 메모리 주소를 참조한다.

console.log(obj1 === obj2); // true

obj2.value = 20;
console.log(obj1.value); // 20 (obj1도 변경된다)
```

## 3. 원시 타입과 객체의 값 비교

자바스크립트의 데이터 타입은 크게 원시 타입과 객체 타입으로 나뉜다.

이 둘의 가장 큰 차이점 중 하나는 변수에 값을 할당하고 비교하는 방식이다.

### 원시 타입 (Primitive Types)

원시 타입(숫자, 문자열, 불리언, null, undefined, 심볼)의 변수를 다른 변수에 할당하면, 메모리 주소 참조를 통해 값 자체를 직접 참조하여 복사한다.

이를 "값에 의한 전달(pass by value)"이라고 한다. 따라서 두 변수는 완전히 독립적이며, 한 변수의 값을 변경해도 다른 변수에 영향을 주지 않는다.

```javascript
let num1 = 10; +let num2 = num1; // num1의 값 10이 복사되어 num2에 할당된다.
console.log(num1 === num2); // true
num2 = 20; // num2의 값을 변경한다.
console.log(num1); // 10 (num1은 변경되지 않는다)
console.log(num2); // 20
```

### 객체 타입 (Object Types)

반면, 객체 타입의 변수를 다른 변수에 할당하면, 메모리 주소(참조)가 복사된다.

이를 "참조에 의한 전달(pass by reference)"이라고 한다.

이 경우 두 변수는 같은 객체를 가리키게 되므로, 한 변수를 통해 객체의 내용을 변경하면 다른 변수에도 영향을 미친다.

```javascript
const obj1 = { value: 10 }; +const obj2 = obj1; // obj1의 참조(메모리 주소)가 복사된다.
console.log(obj1 === obj2); // true (같은 객체를 참조하므로)
obj2.value = 20; // obj2를 통해 객체의 값을 변경한다.
console.log(obj1.value); // 20 (obj1이 참조하는 객체도 변경된다)
```

## 4. ES6 추가 문법

ES6에서는 객체를 더욱 간결하고 편리하게 다룰 수 있는 새로운 문법들이 추가되었다.

### 1) 프로퍼티 축약 표현

객체를 정의할 때, 프로퍼티의 키 이름과 값으로 사용할 변수 이름이 같다면, 변수 이름만으로 프로퍼티를 생성할 수 있다.

```javascript
+const name = "김철수";
const age = 30;
// ES6 이전
// const person = { name: name, age: age };
// ES6 +const person = { name, age };
console.log(person); // { name: '김철수', age: 30 }
```

### 2) 계산된 프로퍼티 이름

객체 리터럴 안에서 대괄호 []를 사용하여 프로퍼티 키를 동적으로 생성할 수 있다.

```javascript
const propName = "age"; +let i = 0; + +const person = {
  name: "홍길동",
  [propName]: 30, // 'age': 30
  [hobby-${++i}]: "축구", // 'hobby-1': '축구'
  [hobby-${++i}]: "농구", // 'hobby-2': '농구'
};

console.log(person); // { name: '홍길동', age: 30, 'hobby-1': '축구', 'hobby-2': '농구' }
```

### 3) 메서드 축약 표현

객체에서 메서드를 정의할 때 function 키워드를 생략할 수 있다.

```javascript
const obj = {
  hello() {
    console.log("안녕?");
  },
};

obj.hello(); // 안녕?
```
