# [08] 제어문 (93p-107p)

  조건에 따라 코드 블록을 실행(조건문)하거나 반복 실행(반복문)할 때 사용
  <br>코드의 실행 흐름을 인위적으로 제어

  순차적인 흐름을 변경하는 것이기 때문에
  <br>가독성을 해치는 단점이 있다

  ### ⬇️

  나중에 살펴볼 
  ```javascript
  forEach()
  map()
  filter()
  reduce()
  ```
  같은 고차 함수를 사용한 함수형 프로그래밍 기법에서는 제어문의 사용을 억제하여 복잡성을 해결하려고 노력한다

  ## 블록문

  0개 이상의 문을 중괄호로 묶은 것
  <br>자체 종결성을 갖기 때문에 블록문의 끝에는 세미콜론을 붙이지 않는다

  ```javascript
  // 블록문
  {
    var foo = 10;
  }

  // 제어문
  var x = 1;
  if (x < 10) {
    x++;
  }

  // 함수 선언문
  function sum(a,b) {
    return a + b;
  }
  ```

  ## 조건문

  주어진 조건식의 평가 결과에 따라 코드 블록의 실행을 결정
  
  (1) if...else문
  <br>(2) switch문


  ### 1. if...else문

  조건식의 평가 결과, 즉 논리적 참 또는 거짓에 따라 실행할 코드 블록을 결정
  
  ```javascript
  if (조건식) {
    // 조건식이 true일 경우 실행
  } else {
    // 조건식이 false일 경우 실행
  }
  ```

  if문의 조건식은 불리언 값으로 평가되어야 한다
  <br> ***조건에 따라 실행될 코드 블록을 늘리고 싶으면 else if문 사용***

  ### 2. switch문

  표현식을 평가하여 그 값과 일치하는 표현식을 갖는 case 문으로 실행 흐름을 옮긴다

  표현식과 일치하는 case문이 없다면
  <br>실행 순서는 default문으로 이동한다(default문은 선택사항)

  ```javascript
  switch (표현식) {
    case 표현식1:
      switch 문의 표현식과 표현식1이 일치하면 실행될 문;
      break;
    case 표현식2:
      switch 문의 표현식과 표현식2가 일치하면 실행될 문;
      break;
    default:
      switch 문의 표현식과 일치하는 case 문이 없을 때 실행될 문;
  }
  ```

  ***주의할 점*** 
  break문이 없다면 case 문의 표현식과 일치하지 않더라도 실행 흐름이 다음 case문으로 연이어 이동한다
  <br>fall through

  만약 if...else문으로 해결할 수 있다면 switch 문보다 if...else 문을 사용하는 편이 좋다
  <br>조건이 너무 많아서 switch 문을 사용했을 때 더 가독성이 좋다면 사용 권장

  ## 반복문

  조건식의 평가 결과가 참인 경우 코드 블록을 실행
  <br>조건식이 거짓일 때까지 반복

  ### 1. for문

  for문은 조건식이 거짓으로 평가될 때까지 코드 블록을 반복 실행
  ```javascript
  for (변수 선언문 또는 할당문; 조건식; 증감식) {
    조건식이 참인 경우 반복 실행될 문;
  }

  for (var i =0; i < 2; i++) {
    console.log(i);
  } // 0 1
  ```

  for문 내에 for문을 중첩해 사용
  ```javascript
  // 두 개의 주사위를 던졌을 때 두 눈의 합이 6이 되는 모든 경우의 수 출력 (이중 중첩 for문 사용)
  for (var i = 1; i <= 6; i++>) {
    for(var j = 1; i <= 6; j++) {
      if (i + j === 6) console.log(`[${i}, ${j}]`)
    }
  } // [1,5] [2,4] [3,3] [4,2] [5,1]
  ```

  ### 2. while문

  주어진 조건식의 평가 결과가 참이면 코드 블록을 계속해서 반복 실행
  
  | 반복문 |     |
  | :----: | :------: |
  | for문 | 반복 횟수가 명확할 때 주로 사용 |
  | while문 | 반복 횟수가 불명확할 때 주로 사용 |

  조건식의 평가 결과가 언제나 참이면 무한루트가 된다
  <br>코드 블록 내에 if문으로 탈출 조건을 만들고 break문으로 코드 블록을 탈출

```javascript
var count = 0;

// 무한루프
while (true) {
  console.log(count); 
  count++;

  // 탈출 조건
  if(count === 3) break;
} // 0 1 2 
```

  ### 3. do...while문

  코드 블록을 먼저 실행하고 조건식을 평가
  <br>따라서 코드 블록은 무조건 한 번 이상 실행

```javascript
var count = 0;

// count가 3보다 작을 때까지 코드 블록을 계속 반복 실행
do {
  console.log(count); // 0 1 2
  count++;
} while (count < 3>);
```

  ## break문

  switch문과 while문에서와 같이 break문은 코드 블록을 탈출한다
  <br>레이블문, 반복문, switch문의 코드 블록 외에 break문을 사용하면 SyntaxError(문법 에러) 발생

  ## continue문

  반복문의 코드 블록 실행을 현 지점에서 중단하고 
  <br>반복문의 증감식으로 실행 흐름을 이동시킨다

  ***break문처럼 반복문을 탈출하지는 않는다***

  
```javascript
var 
// 문자열은 유사 배열이므로 for 문으로 순회할 수 있다
for (var i = 0; i < string.length; i++) {
  // 'l'이 아니면 현 지점에서 실행을 중단하고 반복문의 증감식으로 이동한다
  if (string[i] !== search) continue;
  count++; // continue 문이 실행되면 이 문은 실행되지 않는다
}
```

  ### 1. break문과 continue문의 차이점

  break문은 반복을 즉시 종료하고 반복문 블록 바깥으로 제어를 이동
  <br>continue문은 현재 반복만 중단하고 다음 반복으로 바로 넘어가며 반복 자체는 계속 지속
