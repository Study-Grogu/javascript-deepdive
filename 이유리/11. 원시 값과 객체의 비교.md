# 원시 값과 객체의 비교

### 📍 원시 값

원시 값은 **변경 불가능한 값(immutable value)** 이다. 한번 생성된 원시 값은 읽기 전용 값으로서 변경할 수 없다. 이러한 특성은 데이터의 신뢰성을 보장한다.

- 변수: 하나의 값을 저장하기 위해 확보된 메모리 공간
- 값: 변수에 저장된 데이터, 표현식이 평가되어 생성된 결과
- 상수: 재할당이 금지된 변수, 단 한 번만 할당이 허용되므로 변수 값을 변경(교체)할 수 없음

> 변경 불가능하다는 것은 변수가 아니라 값에 대한 진술이다. 변수는 언제든지 재할당을 통해 변수 값을 변경(교체)할 수 있다.

원시 값을 할당한 변수에 새로운 원시 값을 재할당하면 이전의 원시 값을 변경하는 것이 아니라 **새로운 메모리 공간을 확보**하고 재할당한 원시 값을 저장한 후, 변수는 새롭게 재할당한 원시 값을 가리킨다. 이때 변수가 참조하던 **메모리 공간의 주소**가 바뀐다.

---

### 📍 문자열

문자열은 0개 이상의 문자로 이뤄진 집합이다. 문자열도 원시 값이므로 변경 불가능한 값이다.

- 문자열 타입/2byte: 1개 문자 당 2byte 메모리 공간에 저장
- 숫자 타입/8byte: 어떤 숫자든 동일한 8byte 메모리 공간에 저장

문자열은 **유사 배열 객체**이면서 **이터러블**이므로 배열과 유사하게 각 문자에 접근할 수 있다.

> 유사 배열 객체: 마치 배열처럼 인덱스로 프로퍼티 값에 접근할 수 있고 **length 프로퍼티**를 갖는 객체, **for문 순회** 가능

---

### 📍 값에 의한 전달

```js
let one = 10;
const two = 20;

one = 30;

console.log(one); // 30
console.log(two); // 20
```

변수에 원시 값을 갖는 변수를 할당하면 변수에는 할당되는 변수의 원시 값이 복사되어 전달된다. 이를 **값에 의한 전달**이라고 한다.

```js
const score = 80;

const copy = score;

console.log(score, copy); // 80 80
console.log(score === copy); // true
```

score의 80과 copy의 80은 다른 메모리 공간에 저장된 별개의 값이다. 이후 score 변수의 값을 변경해도 copy 변수의 값에는 어떠한 영향도 주지 않는다.

> 값에 의한 전달은 값을 전달하는 것이 아니라 **메모리 주소**를 전달한다.

---

### 📍 객체

객체는 프로퍼티의 개수가 정해져 있지 않으며, 동적으로 추가되고 삭제할 수 있다. 또한 프로퍼티의 값에도 제약이 없기 때문에 원시 값과 같이 확보해야 할 메모리 공간의 크기를 사전에 정해 둘 수 없다.

---

### 📍 변경 가능한 값

객체는 변경 가능한 값(mutable value)이다.

원시 값을 할당한 변수는 원시 값 자체를 값으로 갖지만, 객체를 할당한 변수가 기억하는 메모리 주소를 통해 메모리 공간에 접근하면 **참조 값**에 접근할 수 있다. 참조 값은 생성된 객체가 저장된 **메모리 공간의 주소 자체**이다. 객체를 할당한 변수를 참조하면 메모리에 저장되어 있는 참조 값을 통해 실제 객체에 접근한다.

객체는 변경 가능한 값이기 때문에 객체를 할당한 변수는 재할당 없이 객체를 직접 변경할 수 있다.(프로퍼티 동적 추가/갱신/삭제 등) 메모리에 저장된 객체를 직접 수정하면, 재할당을 하지 않으므로 객체를 할당한 변수의 참조 값은 변하지 않는다.

객체는 복사해서 생성하는 비용이 크기 때문에 효율적인 메모리 사용을 위해 변경 가능한 값으로 설계되어 있다. 이런 구조적 단점에 따른 부작용으로 원시 값과는 다르게 **여러 개의 식별자가 하나의 객체를 공유**할 수 있다.

#### 얕은 복사와 깊은 복사

```js
const obj = { x: { y: 1 } }; // 중첩된 객체

// 얕은 복사
const c1 = { ...obj };
console.log(c1 === obj); // false: 겉 껍데기는 서로 다른 객체
console.log(c1.x === obj.x); // true: 내부 객체는 공유(같은 참조 값)

// 깊은 복사
console.log(c2 === obj); // false: 완전히 새로운 객체
console.log(c2.x === obj.x); // false: 내부 객체까지 새로 복사됨
```

**[ 얕은 복사 ]**

- 한 단계까지만 복사하는 것
- 객체에 중첩되어 있는 객체의 경우 참조 값을 복사한다.
- 객체를 할당한 변수를 다른 변수에 할당하는 것

**[ 깊은 복사 ]**

- 중첩되어 있는 객체까지 모두 복사하는 것
- 객체에 중첩되어 있는 객체까지 모두 복사하여 완전한 복사본을 만든다.
- 원시 값을 할당한 변수를 다른 변수에 할당하는 것

> 얕은 복사와 깊은 복사로 생성된 객체는 원본과는 다른 객체이다. -> **참조 값이 다른 별개의 객체**

---

### 📍 참조에 의한 전달

객체를 가리키는 변수(원본)를 다른 변수(사본)에 할당하면 원본의 참조 값이 복사되어 전달되는 것을 참조에 의한 전달이라고 한다. (얕은 복사)

원본과 사본은 저장된 메모리 주소는 다르지만 동일한 참조 값을 갖는다. 그러므로 원본과 사본 중 한쪽에서 객체를 변경하면 서로 영향을 주고 받는다.

```js
const person1 = {
  name: "Lee",
};

const person2 = {
  name: "Lee",
};

console.log(person1 === person2); // false: 참조 값이 다름
console.log(person1.name === person2.name); // true: 원시 값
```

- person1과 person2는 내용은 같지만 다른 메모리에 저장된 별개의 객체이다. person1과 person2의 참조 값은 서로 다른 값이다.
- person1.name과 person2.name은 프로퍼티 값을 참조하는데 이는 값으로 평가될 수 있는 표현식이기 때문에 원시 값 'Lee'로 평가된다.
